MODELO DE CALIDAD BOEHM

Aspecto Puntuación Descripción
Características clave del software 5 Identifica las características clave del software que son esenciales para el proyecto. Por ejemplo, podrían ser la gestión de estudiantes, profesores, asignaturas, calificaciones, horarios, etc.
Requisitos de calidad 4.5 Definición detallada de los requisitos de calidad para cada característica. Los requisitos pueden incluir la usabilidad, rendimiento, fiabilidad, seguridad, mantenibilidad y portabilidad, entre otros.
Métricas de calidad 4.5 Define métricas específicas para medir cada uno de los requisitos de calidad. Por ejemplo, el tiempo de respuesta de la interfaz de usuario, la tasa de error, la cantidad de tiempo de inactividad, etc.
Planificación de calidad: 5 Establece un plan para implementar y evaluar la calidad en cada etapa del ciclo de vida del proyecto. Esto podría incluir revisiones regulares, pruebas de rendimiento, pruebas de seguridad, etc.
Diseño y arquitectura 5 Asegúrate de que el diseño y la arquitectura del software estén alineados con los requisitos de calidad definidos. Utiliza patrones de diseño adecuados y mejores prácticas de desarrollo.
Desarrollo 5 Sigue prácticas de desarrollo sólidas para garantizar la calidad del código fuente. Utiliza herramientas de análisis estático para detectar posibles problemas.
Pruebas 4.5 Realiza pruebas exhaustivas para verificar si el software cumple con los requisitos de calidad. Esto puede incluir pruebas unitarias, pruebas de integración, pruebas de regresión y pruebas de aceptación.
Evaluación continua 5 Realiza evaluaciones regulares a lo largo del proyecto para medir el progreso y la calidad en relación con los requisitos establecidos. Ajusta el enfoque según sea necesario.
Capacitación y documentación 5 Asegúrate de que el equipo esté capacitado para seguir las prácticas de calidad y proporciona documentación clara para el desarrollo, las pruebas y la resolución de problemas.
Entrega y mantenimiento 5 Después de la implementación, realiza un monitoreo continuo y responde a problemas de calidad en producción. Realiza actualizaciones y mejoras según sea necesario.

ACCIONES DE MEJORA

  • Revisión continua de los requisitos para su actualización y especificación.
  • Mejora de la gestión de configuración para asegurar la trazabilidad y control de versiones.
  • Realización de evaluaciones periódicas de riesgos y establecimiento de medidas de mitigación.
  • Formalización y trazabilidad de los procesos de verificación y validación.
