HERRAMIENTAS

Criterio Puntuación Razón
Sistema de control de versiones 5 Se utiliza Git para controlar el versionamiento del código y se ha establecido un flujo de trabajo para la colaboración del equipo.
Pruebas 4 Se generaron reports para para realizar pruebas unitarias y HTTP services para la generación de Mocks para pruebas Back-end, sin embargo, se podría mejorar el porcentaje de cobertura de pruebas.
Herramientas para análisis de código 4 Se ha utilizado Pylint para realizar análisis de calidad de código, pero se requiere mejorar la integración con el flujo de trabajo para que los resultados sean más visibles y se tomen acciones en consecuencia.
Herramientas de automatización 5 Se utiliza un WebHub para la integración continua y despliegue automatizado, lo que permite reducir errores humanos y acelerar el proceso de despliegue, sin embargo, se podría mejorar la automatización de pruebas.
Herramientas de gestión de proyectos 5 Se utiliza GitHub Projects para la gestión del proyecto, lo que permite tener un registro de los cambios realizados y las tareas pendientes.

El equipo utiliza herramientas adecuadas para el control de versiones, pruebas, análisis de código, automatización y gestión de proyectos. Aunque existen áreas de mejora, en general las herramientas empleadas son efectivas para el desarrollo del proyecto.
